📊 项目概览

Wishop 是一个多账户微信小店管理桌面应用，支持商品提审、订单管理、违规词检测、淘宝采购自动化等核心功能。每个账户对应一个微信小店，账户上下文在基础设施层解析，业务逻辑与账户无关。

🏗️ 三层分离架构

核心设计原则：Business 层和 API 层不 import Store，依赖通过参数注入。

Infrastructure Layer (account-aware)
store / repositories ipc handlers scheduler preload bridge browser windows logger
职责：解析 accountId → 创建 WxShopClient + scoped logger，注册 IPC 通道，管理持久化数据
Business Layer (account-agnostic)
modules/* services/* jobs/*
职责：纯业务逻辑，通过参数接收 api + logger + config，不直接访问 Store
modules/: task-cycle, violation-detect, fetch-draft-products, delete-failed-products, list-unreviewed-products
services/: taobao-automation-service, order-delivery-service, cloud-sync-service
API Layer (account-agnostic)
wxshop/client.ts wxshop/client-registry.ts
职责：微信小店 API 客户端工厂，每个 accountId 一个实例，独立 token 缓存
Shared Types
accounts orders listing violations scheduling credentials sync license notification global-log errors settings cloud-tasks address-format purchase-status
跨进程共享的类型定义和纯工具函数，按域拆分为独立文件，types.ts 作为 barrel re-export

delete-failed
失败商品删除
状态规则 (statusRules)：editStatus → action 映射。默认：1/72→提审, 3→删除, 其他→跳过。
黑名单检测：product_code 匹配黑名单规则 → 自动跳过。
跳过关键词：标题包含关键词 → 跳过。
限速：提审间隔 3s，删除间隔 1s。
