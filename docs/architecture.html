项目概览

Wishop 是一个多账户微信小店管理桌面应用。每个账户对应一个微信小店，支持商品提审、订单管理、违规词检测、淘宝采购自动化。账户上下文在基础设施层解析，业务逻辑与账户无关。

三层分离架构

核心原则：Business 层和 API 层不 import Store，依赖通过参数注入。

Infrastructure Layer — 账户感知层

解析 accountId，创建 WxShopClient + scoped logger，注册 IPC 通道，管理持久化数据

store / repositories ipc handlers scheduler preload bridge browser windows

Business Layer — 业务逻辑层

纯业务逻辑，通过参数接收 api + logger + config，不直接访问 Store

modules/* services/* jobs/*

API Layer — 接口层

微信小店 API 客户端工厂，每个 accountId 一个实例，独立 token 缓存

wxshop/client.ts wxshop/client-registry.ts

Shared Types — 共享类型层

跨进程共享的类型定义和纯工具函数，按域拆分为独立文件

accounts orders listing violations scheduling settings errors
